1. OVERVIEW & PURPOSE
================================================================================

The Customer Journey system is the core feedback collection engine of Rectangled.io.
It lets businesses create customizable feedback flows (called "Journeys") that
customers interact with after visiting a business. The system captures ratings,
filters positive vs negative experiences, redirects happy customers to leave
Google reviews, and captures detailed feedback from unhappy customers.

WHY IT EXISTS:
- Indian SMBs need more Google reviews to rank higher in local search
- Negative experiences should be captured privately (not on Google)
- Businesses need structured feedback (tags, aspects) not just star ratings
- The whole flow must work from a QR code scan — zero friction

================================================================================
2. WHO USES IT
================================================================================

BUSINESS OWNER / MANAGER (Dashboard User):
- Creates and configures journeys
- Assigns journeys to specific locations
- Generates QR codes for tables/counters/receipts
- Views response analytics
- Sets up automation rules (auto-reply, send coupon, escalate)

CUSTOMER (Public User — No Auth Required):
- Scans QR code or visits journey link
- Rates experience (1-5 stars)
- Gets routed based on rating:
  → High rating (4-5): Redirected to Google review page
  → Low rating (1-3): Shown feedback form with tags
- Optionally provides contact info
- Sees thank you message

SYSTEM / AUTOMATION:
- Triggers post-response actions (coupons, messages, AI replies)
- Creates offline reviews from low ratings
- Tracks customer across journeys

================================================================================
3. WHAT IT DOES
================================================================================

CORE CAPABILITIES:
1. Journey CRUD (create, read, update, archive)
2. Screen builder (drag-reorder, add/delete screens, configure each)
3. 9 screen types (rating, nps, csat, ces, feedback, aspects, etc.)
4. Smart routing based on rating threshold
5. Google review URL auto-fill from GBP connector
6. Location-aware (each journey linked to a business location)
7. QR code generation (PNG/SVG, with logo support)
8. Public survey page (no auth, mobile-first)
9. Response collection & storage
10. Customer creation from responses
11. Offline review creation from low ratings
12. Automation triggers (post-response actions)
13. Analytics on response data

4. HOW IT WORKS (END-TO-END FLOWS)
================================================================================

--- FLOW 1: CREATE JOURNEY ---

Step 1: Owner opens /dashboard/journeys
Step 2: Clicks "Create Journey"
Step 3: Selects type:
        - "Review Option": Filters by rating (low→feedback, high→Google)
        - "Direct Review": Any rating → Google redirect
Step 4: Enters name, selects location from dropdown
Step 5: System creates journey with:
        - Auto-generated slug (format: j-{uuid-slice-10})
        - locationId from selected location
        - Default settings (positiveThreshold=4, enableCoupon=false)
Step 6: System creates 1 default rating screen with embedded config:
        - feedbackTags, redirectLinks, thankYouMessage all in one screen
        - Google review URL auto-filled from GBP connector if location has one
Step 7: Redirected to journey builder for customization

--- FLOW 2: CUSTOMER RATES (REVIEW OPTION TYPE) ---

Step 1: Customer scans QR code → opens /j/{slug}
Step 2: Sees "Please Rate Your Experience" with 5 stars
Step 3: Taps a star rating

  IF RATING >= positiveThreshold (default 4):
    Step 4a: Shows redirect message "Please share your experience on Google!"
    Step 4b: Opens Google review URL in new tab
    Step 4c: Submits response to backend (rating stored)
    Step 4d: Shows thank you message

  IF RATING < positiveThreshold:
    Step 4a: Shows feedback dialog with:
             - Tag pills (Food Quality, Service, Cleanliness, etc.)
             - Textarea for detailed feedback
             - Optional contact fields
    Step 4b: Customer selects tags + writes feedback
    Step 4c: Submits response to backend
    Step 4d: Backend creates offline review (platform='offline', source='journey')
    Step 4e: Shows thank you message
    Step 4f: Automation triggers may fire (send coupon, escalate, etc.)

--- FLOW 3: CUSTOMER RATES (DIRECT REVIEW TYPE) ---

Step 1: Customer scans QR → opens /j/{slug}
Step 2: Sees rating screen
Step 3: Taps any star (positiveThreshold=0, so ALL ratings redirect)
Step 4: Google review page opens
Step 5: Response saved to backend
Step 6: Thank you message shown

--- FLOW 4: RESPONSE SUBMISSION (BACKEND) ---

Step 1: Public page calls journey.submitResponse
Step 2: Backend validates input (journeyId, sessionId, responseData)
Step 3: If customerName/Email/Phone provided:
        - Upsert customer record in customers table
Step 4: Insert journeyResponse with:
        - journeyId, journeyScreenId, customerId, locationId
        - sessionId (browser UUID for dedup)
        - responseData (rating, feedback, tags, aspectTags)
        - ipAddress, userAgent
Step 5: If rating <= 3:
        - Create offline review entry in reviews table
        - platform='offline', source='journey'
        - Includes rating, feedback text, aspect tags
Step 6: Return { success: true, responseId }
Step 7: Automation system checks for matching rules

--- FLOW 5: AUTOMATION EXECUTION ---

Step 1: Response triggers automation check
Step 2: System finds matching rules:
        - journey_completed_positive (rating >= threshold)
        - journey_completed_negative (rating < threshold)
        - journey_abandoned (session exists, no response)
Step 3: For each matching rule, create queue entry:
        - scheduledFor = now + rule.delayMinutes
Step 4: Background processor picks up queue items where scheduledFor <= now
Step 5: Executes action based on actionType:
        - send_coupon: Issue coupon from template
        - send_message: Email/WhatsApp via templates
        - create_escalation: Flag for manager review
        - tag_customer: Add tags to customer record
        - trigger_journey: Start another journey
        - ai_reply_review: Generate AI reply via OpenRouter → post to Google
Step 6: Update queue status (completed/failed)

{
  aspects: ['Food', 'Service', 'Ambience', 'Value', 'Cleanliness']

Purpose: Redirect to external review platforms (legacy, used in 4-screen flows)

{
  message: 'We appreciate your feedback! Would you mind sharing your experience?',
  links: [{ platform: 'Google', url: '' }]

Purpose: Freeform text feedback collection
Default config:
{
  placeholder: 'Tell us more about your experience...'
}

--- CONTACT_COLLECTION ---
Purpose: Collect customer contact details
Default config:
{
  fields: ['name', 'email', 'phone']
}

--- THANK_YOU ---
Purpose: End-of-journey thank you message (legacy, used in 4-screen flows)
Default config:
{
  message: 'Thank you for your feedback!',
  showCoupon: false
}

--- NPS (Net Promoter Score) ---
Purpose: 0-10 scale "How likely to recommend"
Default config:
{
  question: 'How likely are you to recommend us?',
  lowLabel: 'Not at all likely',
  highLabel: 'Extremely likely'
}

--- CSAT (Customer Satisfaction) ---
Purpose: Satisfaction measurement
Default config:
{
  question: 'How satisfied are you?',
  scaleType: 'stars'                               // 'stars' | 'emojis' | 'numbers'
}

--- CES (Customer Effort Score) ---
Purpose: 1-7 scale effort measurement
Default config:
{
  question: 'How easy was your experience?',
  lowLabel: 'Very difficult',
  midLabel: 'Neither',
  highLabel: 'Very easy'

10. DEPENDENCIES & INTEGRATIONS
================================================================================

--- LOCATIONS ---
Relationship: journeys.locationId → locations.id (nullable FK, cascade delete)
Usage:
  - Filter journeys by location in list page
  - Find matching GBP connector for Google review URL
  - Store locationId in journey responses
  - QR code can include ?loc={locationId} parameter

--- GBP CONNECTOR (Google Business Profile) ---
Relationship: Indirect via locationId matching
Usage:
  - connectorInstances where connectorTypeId='gbp' AND locationId matches
  - Extracts config.placeId from connector
  - Generates: https://search.google.com/local/writereview?placeid={placeId}
  - Auto-fills in: journey creation, screen save, builder load
  - Also used by automation (ai_reply_review) to post replies to Google

--- CUSTOMERS ---
Relationship: journeyResponses.customerId → customers.id (nullable FK, set null)
Usage:
  - Created during submitResponse if name/email/phone provided
  - Upserted by email match within workspace
  - Tracks: firstSeenAt, lastSeenAt, totalReviews, averageRating
  - Can be tagged by automation rules

--- REVIEWS ---
Relationship: reviews.journeyResponseId (no FK constraint, stored as reference)
Usage:
  - Low ratings (<=3) create offline review:
    platform='offline', source='journey'
    platformReviewId='offline-{responseId}'
    Includes rating, feedback text, aspect tags
  - Shows in Inbox alongside Google/Zomato reviews
  - Can trigger 'review_posted' automation

--- COUPONS ---
Relationship: couponInstances.journeyResponseId → journeyResponses.id
Usage:
  - showCoupon flag in thank_you / rating screen config
  - Automation action 'send_coupon' issues coupon after response
  - Coupon template linked via actionConfig.couponTemplateId

--- AUTOMATION ---
Relationship: automationRules.journeyId → journeys.id (nullable FK, cascade)
Usage:
  - Rules can be journey-specific or global (journeyId=null)
  - Triggers: journey_completed_positive/negative, journey_abandoned
  - Queue entries reference journeyResponseId
  - Default rules seeded on journey creation (optional)

--- QR CODES ---
Relationship: Generated on-demand, not stored
Usage:
  - QR encodes URL: {baseUrl}/j/{slug}?loc={locationId}
  - Generated via 'qrcode' npm package
  - Supports PNG (default 300px) and SVG
  - High error correction for logo overlay variant
  - Bulk generation for multiple journeys/forms

--- ANALYTICS ---
Relationship: journeyResponses queried for metrics
Usage:
  - Response count per journey
  - Rating distribution
  - Conversion rate (responses / QR scans)
  - Tag/aspect frequency analysis
  - NEV (Net Emotional Value) scores from responses
  - CLI (Customer Loyalty Index) segments

--- NEV (Net Emotional Value) ---
Relationship: nevResponses.journeyResponseId → journeyResponses.id
Usage:
  - Emotional sentiment scoring from journey responses
  - Separate analysis layer on top of ratings

--- CLI (Customer Loyalty Index) ---
Relationship: cliResponses.journeyResponseId → journeyResponses.id
Usage:
  - Customer loyalty segment tracking
  - Derived from journey response patterns

11. AUTOMATION SYSTEM (DETAILED)
================================================================================

PURPOSE:
Automate post-response actions — send coupons to unhappy customers,
AI-reply to Google reviews, escalate critical feedback, etc.

TRIGGER EVENTS:
1. journey_completed_positive  — Rating >= positiveThreshold
2. journey_completed_negative  — Rating < positiveThreshold
3. journey_abandoned           — Session started but no response submitted
4. review_posted              — Any review posted (from any source)
5. review_posted_google       — Google review specifically
6. customer_dormant           — Customer inactive for X days
7. custom                     — Manual/API trigger

ACTION TYPES:
1. send_coupon       — Issue coupon from template to customer
2. send_message      — Email or WhatsApp message via templates
3. create_escalation — Create escalation ticket for manager
4. tag_customer      — Add tags to customer record
5. trigger_journey   — Start another journey for the customer
6. ai_reply_review   — Generate AI reply and post to Google

DEFAULT RULES (seeded on journey creation):
1. "Thank positive reviewers" — journey_completed_positive, 3 day delay, send_message
2. "Win-back negative reviewers" — journey_completed_negative, 1 day delay, send_coupon
3. "Remind abandoned journeys" — journey_abandoned, 2 hour delay, send_message

AI REPLY EXECUTION FLOW:
1. Queue item picked up by background processor
2. Load review from DB (using reviewId)
3. Check ratingFilter (positive=4-5, negative=1-3, all)
4. Generate reply via OpenRouter API (gpt-4o-mini):
   - System prompt with tone, business name, max length
   - Review text as user input
5. Fetch GBP connector with valid access token (refresh OAuth if expired)
6. Post reply to Google via GBP adapter using review.metadata.reviewResourceName
7. Update queue status to completed/failed

QUEUE PROCESSING:
- Background interval checks for pending items where scheduledFor <= now
- Processes one at a time
- Max attempts tracking
- Error messages stored in lastError
- Status transitions: pending → processing → completed/failed
- Can be manually cancelled (pending only)

12. QR CODE SYSTEM
================================================================================

GENERATION:
- Library: 'qrcode' npm package
- Default: PNG format, 300px, error correction level M
- Logo variant: High error correction, 2px margin (for center logo overlay)
- SVG support available

URL FORMAT:
  {NEXT_PUBLIC_APP_URL}/j/{journey.slug}
  Optional: ?loc={locationId}

- feedbackTags from feedbackScreen.config.tags

RATING INTERACTION:
- Star hover effect (highlights stars up to hovered)
- Click locks in rating
- After click, evaluates routing:

  IF positiveThreshold = 0 (Direct Review):
    → Redirect to first Google link in redirectLinks
    → Submit response with rating
    → Show thank you

  IF rating >= positiveThreshold:
    → Show redirect message
    → Open Google review URL (new tab)
    → Submit response with rating
    → Show thank you

  IF rating < positiveThreshold:
    → Show feedback dialog modal
    → Display tag pills for selection (tappable)
    → Show textarea with placeholder
    → On submit:
      - Call journey.submitResponse with rating + feedback + tags
      - Close dialog
      - Show thank you message

THANK YOU SCREEN:
- Shows thankYouMessage text
- Optional coupon display (if showCoupon enabled)
- No further navigation
